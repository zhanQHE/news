# 发布检查清单

这份清单用于在更新仓库内容前做一次快速确认，避免文档、脚本和配置之间出现明显脱节。

## 基础检查

- 确认 README 中的启动方式仍然可用。
- 检查 `requirements.txt` 与 `pyproject.toml` 是否描述了同一组核心依赖。
- 确认 `start-http.sh` 与 `start-http.bat` 指向的服务入口一致。
- 如果修改了通知渠道，检查对应的配置示例是否同步更新。

## 文档检查

- 查看 `docs/index.html` 是否还能正常打开。
- 如果新增 FAQ 或使用说明，确认中英文文档没有互相矛盾。
- 避免把临时输出、调试日志或本地路径写进公开文档。

## 发布后观察

- 拉取远端仓库，确认本地工作区保持干净。
- 查看 GitHub 页面中的 README 渲染效果。
- 记录下一轮需要补充的维护项。
